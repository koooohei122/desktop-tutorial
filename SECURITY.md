# Security Policy

## Supported versions

The `main` branch and the latest release tag are supported.

## Reporting a vulnerability

Please report vulnerabilities through GitHub Security Advisories:

1. Go to the repository Security tab.
2. Create a private vulnerability report.
3. Include reproduction details, impact, and suggested mitigation.

Avoid creating a public issue for security vulnerabilities.

## Response expectations

- Initial acknowledgment: within 5 business days
- Triage and impact assessment: as soon as practical
- Fix and coordinated disclosure: based on severity and complexity
